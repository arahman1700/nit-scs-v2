Add `.max()` constraints to every `z.string()` field that lacks one across ALL backend Zod schemas.

**Length conventions (apply consistently):**
- Codes (itemCode, warehouseCode, poNumber, etc.): `.max(50)`
- Names (itemName, supplierName, projectName, etc.): `.max(255)`
- Short text (driverName, vehiclePlate, contactPhone, etc.): `.max(100)`
- Descriptions/notes/comments: `.max(2000)`
- Addresses: `.max(500)`
- URLs (trackingUrl, googleMapsPickup, etc.): `.max(2000)`
- Reasons/resolutions: `.max(1000)`
- Email template bodies: `.max(10000)`

**Method:**
1. Search each file for `z.string()` calls that do NOT chain `.max()`, `.uuid()`, `.email()`, `.datetime()`, or `.url()` (those are already bounded by format).
2. For every bare `z.string().optional()` or `z.string().min(N)` without `.max()`, insert an appropriate `.max()` based on the field name and the conventions above.
3. For `z.string()` inside `z.array(z.string())`, add `.max(100)` to each element.
4. Do NOT change fields that already have `.max()` -- preserve existing limits.
5. Do NOT change fields typed as `.uuid()`, `.email()`, `.datetime()`, `.url()` -- these are format-bounded.
