From packages/backend/src/domains/inbound/services/grn.service.ts:
- `create(headerData, lines, userId)` -> creates GRN in 'draft' status
- `submit(id)` -> transitions draft -> pending_qc, auto-creates QCI/DR if needed
- `approveQc(id, userId)` -> transitions pending_qc -> qc_approved
- `receive(id, userId?)` -> transitions qc_approved -> received
- `store(id, userId)` -> transitions received -> stored, calls addStockBatch()

From packages/backend/src/domains/outbound/services/mi.service.ts:
- `create(headerData, lines, userId)` -> creates MI in 'draft' status
- `submit(id)` -> transitions draft -> pending_approval
- `approve(id, action, userId, comments?, io?)` -> processes approval, calls reserveStockBatch()
- `issue(id, userId, partialItems?)` -> transitions approved -> issued, deducts stock

From packages/backend/src/domains/outbound/services/mrn.service.ts:
- `create(headerData, lines, userId)` -> creates MRN in 'draft'
- `approve(id, action, userId)` -> processes approval, adds stock back

From packages/backend/src/domains/outbound/services/mr.service.ts:
- `create(data, userId)` -> creates MR in 'draft'
- `submit(id)` -> transitions draft -> pending_approval
- `approve(id, action, userId)` -> processes approval

From packages/backend/src/domains/transfers/services/stock-transfer.service.ts:
- `create(data, userId)` -> creates WT in 'draft'
- `submit(id)` -> transitions draft -> pending_approval
- `approve(id, action, userId)` -> processes approval
- `complete(id, userId)` -> transitions to completed, deducts from source, adds to destination

From packages/backend/src/domains/inbound/services/qci.service.ts:
- `create(data, userId)` -> creates QCI linked to GRN
- `inspect(id, data, userId)` -> transitions pending -> inspected
- `complete(id, userId)` -> transitions inspected -> completed

From packages/backend/src/domains/inbound/services/dr.service.ts:
- `create(data, userId)` -> creates DR linked to GRN
- `resolve(id, data, userId)` -> transitions open -> resolved
