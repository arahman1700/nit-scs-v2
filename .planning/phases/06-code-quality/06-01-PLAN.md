    Part A -- StatusFlow Zod Schema:

    Create `packages/backend/src/domains/system/schemas/dynamic-document-type.schema.ts` with:

    1. `statusFlowSchema` -- a Zod schema matching the StatusFlowConfig interface:
       ```
       {
         initialStatus: z.string().min(1).max(50),
         statuses: z.array(z.object({
           key: z.string().min(1).max(50),
           label: z.string().min(1).max(100),
           color: z.string().min(1).max(30),
         })).min(1, 'At least one status is required'),
         transitions: z.record(z.string(), z.array(z.string())),
       }
       ```

    2. Add `.refine()` checks:
       - `initialStatus` must exist as a key in the `statuses` array
       - All transition keys (source statuses) must exist in `statuses`
       - All transition values (target statuses) must exist in `statuses`
       - Detect circular transitions: use DFS from initialStatus to find if at least one path reaches a terminal status (a status with no outgoing transitions). If every path from initialStatus cycles back without reaching a terminal, reject.

    3. `createDocumentTypeSchema` and `updateDocumentTypeSchema` for the full input types, with statusFlow optional and validated when present.
